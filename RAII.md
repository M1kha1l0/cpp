Принцип программирования, в котором захват и управление ресурсом ограничивается рамками жизни объекта.

Рассмотрим пример:

```cpp
void g(int y) {
	if(y == 0) {
		throw 1;
	}
}

void f(int x) {
	int* p = new int(x);
	g(*p);
	delete p;
}

int main() {
	f(0);
}
```

Вроде как код написан корректно, управление динамической памятью не содержит ошибок, но исключение выкинутое в функции g приводит к утечке памяти. delete за пользователя никто вызывать не будет. Поэтому для решения подобного рода проблем существует 3 решения:
- Не использовать исключения вообще.
- Использовать try catch.
- Решить проблему с помощью объектов.
Вариант 1 не очень подходит, не хочется ограничивать возможности языка. Второй вариант очень дорогой и использовать часто не очень хорошо. Остается только 3 вариант - написать объект, который будет управлять ресурсом автоматически.

```cpp
struct Wrapper {
	int* ptr;
	Wrapper(int* p): ptr(p) {}
	~Wrapper() {
		delete ptr;
	}
	
	int& operator*() {
		return *ptr;
	}
	
	const int& operator*() const {
		return *ptr;
	}
};
```

Используя эту структуру код будет без утечек памяти даже с исключением:

```cpp
void g(const Wrapper& p) {
	if(*p == 0) {
		throw 1;
	}
}

void f(int x) {
	Wrapper w(new int(x));
	g(w);
}

int main() {
	f(0);
}
```

Такая структура называется SmartPtr. В C++ это unique_ptr. Есть и другие виды, подробнее здесь: [[(!)Смарт поинтеры]].
Мы же разберем только реализацию SmartPtr:

```cpp
template <typename T>
class SmartPtr {
	T* ptr_;
	
	public:
	SmartPtr(T* ptr): ptr_(ptr) {}
	SmartPtr(const SmartPtr& other) = delete;
	SmartPtr& operator=(const SmartPtr& other) = delete;
	~SmartPtr() {
		delete ptr_;
	}
	
	T& operator*() {
		return *ptr_;
	}
	
	const T& operator*() const {
		return *ptr_;
	} 
	
	T* operator->() {
		return ptr_;
	}
	
	const T* operator->() const {
		return ptr_;
	}
};
```

Обратим внимание, что здесь удалено копирование. Это нужно за тем, чтобы избежать двойного освобождения памяти по одному и тому же указателю.

Данный класс автоматически будет вызывать деструктор для освобождение динамического ресурса после выхода из скоупа.
