# Указатели, массивы, строки

## Указатель
Указатель - переменная, диапазон значений, которой состоит из адресов ячеек памяти и специального значения (нулевого адреса).

Основное [правило](http://unixwiz.net/techtips/reading-cdecl.html) чтения указателя: от названия переменной читай влево пока возможно, а дальше читай вправо.

У любой переменной любого типа данных есть определенная ячейка в памяти, в которой хранится значение.  Поскольку адрес в памяти это номер, то указатель по сути своей представляется числом и занимает **8 байт**, но не на всех системах (на 32-битных системах 4 байта). При этом размер не зависит от типа указателя ибо адрес в памяти не зависит от типа, тип указателя определяет только размер одного блока в памяти.

Оператор "&" позволяет узнать по какому адресу в памяти лежит переменная. Адрес является константой, т.е. адрес переменной невозможно изменить.
Также "&" позволяет работать с любой переменной в независимости от типа данных. Но есть важный нюанс: оператор работает только с **rvalue** значениями,  нельзя получить адрес от промежуточных вычислений, т.е. невозможна ситуация:

```cpp
int a = 10;
std::cout << &(&a+1);
// или
std::cout << &a++;
```

Но вполне корректная:
```cpp
int a = 10;
std::cout << &(++&a);
```

Пример использования:
```cpp
#include <iostream>

int main() {
	int x = 10;
	std::cout << &x << '\n';
	// Примерный вывод: 0x7ffd0da7147c 
	return 0;
}
```

Важно ! Когда мы хотим объявить несколько указателей на одной строке, то перед каждой переменной указателя ставится звездочка:

```cpp
int *x, *y; // корректно, это все указатели
int* x, y; // не то что нужно, x - указатель, а y - переменная типа int
```

Для каждого типа определен тип данных, в котором хранится это специальное значение адреса в памяти.
Например:
```C++
int x = 10;
int* p = &x; // &: T -> T*
```

Для типа int адрес ячейки будет соответствовать типу int*, char - char* и т.д.
В данном случае 'звездочка' это часть типа.

Для оператора "&" есть и обратный оператор "звездочка" - разыменование указателя. После разыменования возвращается [[#^e608ce|ссылка]] на разыменованный объект.

```cpp
#include <iostream>

int main() {
	int x = 20;
	int *y = &x;
	int z = *y;
	
	// выведет 20, т.к. переменная хранит в себе число 20
	std::cout << "x = " << x << '\n';
	// выведет адрес в памяти x, т.к. указатель хранит адрес
	std::cout << "y = " << y << '\n';
	// выведет 20, т.к. разыменованный указатель возвращает значение в памяти
	std::cout << "z = " << z << '\n';
	
	return 0;
}
```

Но важно, при объявлении адреса **y** - 'звездочка' это часть типа. А **'звездочка'y** это унарная операция, которая возвращает переменную типа, адрес которого хранится в **y**.

При инициализации указателя ему выдается *нулевой адрес (nullptr)*.  Это адрес зарезервированный под "ничего" и используется только для обозначения пустого указателя.

#### Висячие указатели
Такими называются указатели, которые хранят в себе адрес переменной жизнь, которых закончилась, и если мы хотим разыменовать указатель с адресом удаленной переменной, то это UB

```cpp
#include <iostream>

// компилируем с флагом -O2 для оптимизации компилятором

int main() {
	int x = 20;
	int *y = &x;
	{
		int b = 2;
		y = &b;
	}
	
	std::cout << *y << '\n'; // 0
	
	return 0;
}
```

Также опасно делать касты статических переменных к указателю другого типа, это UB.
Например:

```cpp
int z = 342342;
std::cout << *(double*)&z << '\n';
```

#### Вложенные указатели
Указатели могу быть вложенными, т.е. указатели могут указывать на указатели

```cpp
int x = 10;
int* p = &x;
int** pp = &p;
```

Здесь **pp** указывает на адрес указателя **p** поскольку это тоже число, следовательно оно  где-то лежит в памяти. Правила разыменования такие же. Если разыменовать **pp**, то получим адрес **p**, а если уже дважды разыменовать **pp**, то будет уже значение **x**.

```cpp
#include <iostream>

int main() {
	int x = 10;
	int* p = &x;
	int** pp = &p;
	
	std::cout << p << " = " << *pp << '\n';
	std::cout << *p << " = " << **pp << '\n'; // оба разыменованы в x
	std::cout << p << " != " << pp << '\n';
	
	return 0;
}
```


### Указатель на функцию
Когда Берется указатель на функцию, то на самом деле берется адрес из машинного кода на начало области памяти, где записана эта функция. Важно, чтобы функция была определена, иначе без определения не генерируется машинный код (будет ошибка линковки).

Как создать и передать указатель на функцию:
```cpp
#include <iostream>

bool cmp(int x, int y) {
	return x > y;
}

int main() {
	bool (*p)(int, int) = &cmp;
	int x = 11, y = 9;
	if(p(x, y)) {
		// такой  вывод
		std::cout << "x > y";
	}
}
```

В начале указывается возвращаемый тип, после задается имя указателя на функцию в скобках, а дальше в скобках типы принимаемых значений.
Также как и переменные функции имеют свой адрес в памяти, поэтому когда мы хотим в указатель на функцию присвоить адрес другой функции, то используем оператор **&**.
Но можно не использовать **&**, функция неявно приведется в адрес.

В случае если есть перегрузки функций, то при явном указании типа компилятор разрешает кому какой указатель присвоить

```cpp
bool f(int) {}
bool f(double) {}

int main() {
	// оба выражения корректны и получим то, что хотели
	// для double получим f(double)
	// для int получим f(int)
	bool (*p1)(int) = &f;
	bool (*p2)(double) = &f;
	
	// &f; а вот это выражение некорректно, т.к. появляется неопределенность
	// компилятор не знает к какой функции обратиться
}
```

Еще можно делать массив из указателей на функцию:

```cpp
void (*p[10])(int); // массив из 10 указателей на на функцию void(int)
```

А если хочется, чтобы функция возвращала указатель на функцию, то

```cpp
void (*(*p[10]))(int);
```


### Арифметика указателей
Указатели поддерживают арифметику целых чисел, т.е. к указателю можно прибавить любое целое число тем самым сдвинув его на несколько ячеек размера типа указателя.
Но вообще любая арифметическая операция над указателем это почти UB.

```cpp
int* p;
p + 1; // сдвитется на один размер int
```

В данном случае указатель сдвинулся на размер типа int. С любым другим указателем история та же (char сдвигается на размер char и т.д.).

Поддерживаются все арифметические операции (НО только сложение и вычитание и любой их унарный оператор)

```cpp
#include <iostream>

int main() {
	int arr[3] = { 1, 2, 3 };
	int* p = &arr[1];
	
	std::cout << *(p+1) << '\n'; // 3
	std::cout << *(p-1) << '\n'; // 1
	std::cout << *p << '\n'; // 2
	
	return 0;
}
```


Также поддерживается только операция вычитания двух указателей одинаковых типов. 

```cpp
#include <iostream>

int main() {
	int arr[3] = { 1, 2, 3 };
	int* p1 = &arr[0];
	int* p2 = &arr[2];
	
	std::cout << p1 - p2 << '\n'; // -2
	
	return 0;
}
```

Такое вычисление дает понимание сколько ячеек памяти определенного типа данных лежит между двумя указателями.

### Операторы сравнения

Для указателей определены операции сравнения. Сравниваются просто как числа.
Но опять же указатели должны быть одинаковых типов.

### void*
Это специальный тип указателя, когда нужен только адрес ячейки памяти. Для void* недоступны операции разыменования и арифметики. Любой тип указателя может неявно кастится к void*.

### nullptr
Это специальный тип указателя имеющий тип **nullptr_t**, который является константным указателем и неявно приводится к любому типу указателей.
В отличии от NULL в C - nullptr имеет тип указателя, а NULL это целочисленная константа. По смыслу они означают одно и то же, но рекомендуется использовать nullptr.

### Полезные свойства
Указатели позволяют менять значение переменных из других скоупов. Например функция **swap**.

```cpp

swap(int *a, int *b) {
	int t = *a;
	*x = *y;
	*y = t;
}
```

## Ссылка
Когда необходимо работать не с копией, а непосредственно с объектом, который передали в аргументы. По сути своей ссылка это псевдоним для создания не копии объекта, а другого имени для объекта (т.е. любой объект и его псевдоним взаимозаменяемые). Псевдонимы работают для любых типов, хоть для указателей (но указателей на ссылки не бывает). ^e608ce

```cpp
// будет создан псевпоним y для переданного аргумента
void f(int& y) {
	++y;
}

int x = 10;
f(x);
```

Если мы будем присваивать псевдониму значение другой переменной, то это как изначально в псевдоним поставить исходный объект:

```cpp
int x = 10;
int& y = x;
int& t = y; // ссылки приравниваются
int z = 20;
y = z; // x = y = t = z = 20
```

Ссылки нельзя оставлять без инициализации, простое объявление приводит к ошибке компиляции. А также ссылки всегда должны быть инициализированы значением **lvalue**

Формально **int** и **int&** являются разными типами поэтому можно перегружать функции с такими параметрами, но когда мы захотим туда передать объект, то будет ошибка компиляции ибо компилятор не знает, что лучше выбрать (нет разницы как вызываться от объекта или от его псевдонима):

```cpp
void f(int& x) {
	++x;
}

void f(int x) {
	++x;
}

int main() {
	int a = 10;
	f(a); // ошибка компиляции
}
```

Важно понимать, что ссылка при очистке не уничтожает объект, она лишь перестает быть его псевдонимом, но когда уничтожается объект, то обращение по ссылке к нему - UB.

```cpp
#include <iostream>

int& foo() {
	int i = 10;
	// после завершения функции i удаляется, значит объкта больше нет
	return i;
}

int main() {
	int& i = foo();
	std::cout << i; // UB
	return 0;
}
```

На уровне компилятора передача объекта по ссылке это передача указателя на него.
В отличии от указателей нельзя создать ссылку на ссылку, будет либо ошибка компиляции, либо неявно приведется к обычной ссылке.

Важный пример:

```cpp
int& g() {
	int* p = new int(1);
	return *p;
}

int main() {
	int& x = g();
	delete &x;
}

// некорректная версия
int main() {
	int x = g(); // тут вернется копия значения
	delete &x;   // вернулась копия => нет права удаления из стека
}
```

 Мы уже знаем, что разыменование указателя возвращает ссылку на переменную, в main создавая псевдоним разыменованного указателя мы приравниваем x значение и адрес созданного указателя, а значит, что удаление по адресу x будет корректно (это тот же адрес указателя **p**).

 То же самое будет, если мы будем возвращать из функции ссылку на static переменную, т.к. она лежит в памяти на протяжении всей жизни программы.
## Массив

Как объявляется и инициализируется массив массив:

```cpp
int main() {
	// объявлен, но не инициализирован 
	int a[5]; // массив размера 5, не гарантируется, что внутри 0
	
	int b[5] = { 1, 2, 3, 4, 5 }; // инициализированный массив
	int c[] = { 1, 2, 3, 4, 5, 6 }; // инициализация также задает его размеры
	int d[5] = {}; // инициализируется нулями
	int e[5] = { 1, 2 }; // остальная часть массива будет заполнена нулями
	
	int f[5]{}; // можно не писать равно и будет то же самое
}
```

К элементам массива обращаются через оператор []:

```cpp
#include <iostream>

int main() {
	int a[5]{};
	a[4] = 1;
	
	for(int i = 0;i < 5;i++) {
		std::cout << a[i];
	}
	// вывод 00010
}
```

При этом нет никакой проверки выхода за границы массива, а непосредственно выход за границу это UB.

По сути своей массив это непрерывный блок памяти хранящий только указатель на начала этого блока в памяти.
Т.е. по сути обращение к элементу массива это сдвиг указателя на определенное число байт в зависимости от размера типа.

```cpp
#include <iostream>

int main() {
	int a[5]{};
	a[4] = 1;
	
	std::cout << a[4] << " = " << *(a + 4) << '\n';
}
```

Когда мы так делаем наш массив неявно преобразуется к указателю.
Это свойство позволяет массивам и указателям быть взаимозаменяемыми

```cpp
#include <iostream>

int main() {
	int a[5]{ 1, 2, 3, 4, 5 };
	a[4] = 1;
	int* ptr = a + 2;
	
	std::cout << a[1] << " = " << ptr[-1] << " = " << *(ptr - 1) << '\n';
}
```

В данном случае `ptr[-1]` - корректное выражение, поскольку указатель не на начале массива, а значит к предыдущим элементами обращаться можно, но опять же, если выйти за границы слева или справа - это UB.
Также указатель может менять значения в этом массиве по любому доступному индексу.

Как мы убедились на предыдущих примерах оператор [] на самом деле не совсем оператор, а скорее часть синтаксиса для удобства чтения, ибо разрешена и такая ситуация:

```cpp
#include <iostream>

int main() {
	int a[5]{ 1, 2, 3, 4, 5 };
	// записи эквивалентны
	std::cout << 2[a] << " = " << *(2 + a) << '\n';
}
```

НО ЕСТЬ ВАЖНОЕ ОТЛИЧИЕ ОТ ОБЫЧНЫХ УКАЗАТЕЛЕЙ
-  Запрещен оператор присваивания. Нельзя присвоить одному массиву другой, это выглядит как присваивание по сложности O(1), но на деле неожиданное O(n).
-  Нельзя использовать унарные операторы для массивов.
-  Оператор `sizeof` работает по-разному для указателя и для массива. `sizeof(ptr)` выведет всегда 8 байт, а `sizeof(array)` выведет сколько байт занимает весь массив.
-  При передаче в функцию массив всегда неявно преобразуется к указателю. Поэтому нельзя перегрузить функцию, которая принимает указатель, функцией, которая принимает массив фиксированной длины.

### VLA Массивы
Массивы переменной длинны.
```cpp
#include <iostream>
int main() {
	int n;
	std::cin >> n;
	int a[n];
}
```

Изначально введено в C, и поддерживается для обратной совместимости некоторыми компиляторами, но в стандарте запрещено использовать такие конструкции.

### Двумерные массивы
Определяются как указатель на массив из указателей.

```cpp
int a[5][5]; // двумерный массив. 5 указателей на 5 указателей
int *b[5];   // массив из пяти указателей на int
int (*c)[5]; // а вот это уже указатель на массив из 5 int
```

При передаче в функцию 1 и 2 типы это одно и то же, а вот 3 вариант это уже другое.
## Строки
Строки это те же массивы, но не произвольного типа, а `char*`. 
В конце каждой строки стоит специальный символ конца строки `\0`.
