О работе с ссылками: [[Пространства имен и перегрузка функций]]

# R-value reference
Это ссылка, которая не имеет адреса в памяти и псевдонима. Как правило все временные объекты (результаты вычислений, результаты функций) являются r-value ссылками на объект. Получило свое название из-за того, что находится справа от выражения присваивания.

Такие ссылки обладают единственностью и имеют тип `Type&&`.

```cpp
#include <iostream>

int main() {
	int&& rv = 10
	return 0;
}
```

Важный момент: при передаче в аргументы r-value ссылки она становится l-value.

```cpp
#include <iostream>

struct A {};

void g(A& x) {
	std::cout << "1\n";
}

void g(A&& x) {
	std::cout << "2\n";
}

void f(A&& x) {
	g(x);
}

int main() {
	f(A{});
}
```

Это связано с тем, что переданные аргументы в функцию сохраняются в области памяти (хоть и временной) соответственно имеют адрес в памяти, а значит r-value не являются.
# Move-семантика
Идея такая: каждый раз вызывать оператор копирования при передаче, например, временного объекта очень дорого по памяти (двойное выделение памяти):

```cpp
#include <iostream>

class BigData {
	public:
	BigData(const BigData& other) {
		std::cout << "Create BigData\n";
	}
	BigData() {
		std::cout << "Create BigData\n";
	}
};

class A {
	BigData* d;
	public:
	A(): d(new BigData) {}
	A(const A& other): d(new BigData) {}
};

class B {
	A field;
	
	public:
	B(const A& other): field(other) {}
};

int main() {
	B b(A{});
}
```

Но по идее нам не нужно 2 раза выделять память, чтобы с помощью временного объекта проинициализировать новый.
Чтобы убрать данное недоразумение исправим копирование в классе A:

```cpp
#include <iostream>

class BigData {
	public:
	BigData(const BigData& other) {
		std::cout << "Create BigData\n";
	}
	BigData() {
		std::cout << "Create BigData\n";
	}
};

class A {
	BigData* d;
	public:
	A(): d(new BigData) {}
	A(A& other): d(other.d) { other.d = nullptr; }
};

class B {
	A field;
	
	public:
	B(A& other): field(other) {}
};

int main() {
	A a;
	B b(a);
}
```

Теперь все работает за одну операцию выделения памяти, но теперь мы не можем хранить 2 копии объекта и передавать временные объекты в качестве аргумента конструктора. Это беда.

Поэтому были придуманы **r-value** ссылки. По сути своей это флаг для компилятора, что мы хотим сделать перемещение ресурса, а не копирование.

```cpp
#include <iostream>

class BigData {
	public:
	BigData(const BigData& other) {
		std::cout << "Create BigData\n";
	}
	BigData() {
		std::cout << "Create BigData\n";
	}
};

class A {
	BigData* d;
	public:
	A(): d(new BigData) {}
	A(const A& other): d(other.d) { other.d = nullptr; }
};

class B {
	A field;
	
	public:
	B(A& other): field(other) {}
};

int main() {
	A a;
	B b(a);
}
```

На помощь приходит `std::move`, эта функция кастит l-value ссылки к r-value и тогда все работает:

```cpp
#include <iostream>

class BigData {
	public:
	BigData(const BigData& other) {
		std::cout << "Create BigData\n";
	}
	BigData() {
		std::cout << "Create BigData\n";
	}
};

class A {
	BigData* d;
	public:
	A(): d(new BigData) {}
	A(A& other): d(other.d) { other.d = nullptr; }
};

class B {
	A field;
	
	public:
	B(A& other): field(other) {}
};

int main() {
	A a;
	B b(a);
}
```
