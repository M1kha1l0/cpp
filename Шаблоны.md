Идея шаблонов: нужно писать код, который одинаково работает для произвольных типов (мета-переменных типа). Это парадигма программирования.

# Объявление
Для шаблонов некоторая переменная **T** (тип) является именем мета-переменной. Нужно понимать, что для вызовов для разных типов генерируется разный ассемблерный код.

Шаблонными могут быть и функции и классы.
Пример использования шаблона функции:

```cpp
// не очень эффективная реализация
// из-за возможного нетривиального копирования
// модифицируем позже
template<typename T>
void swap(T& x, T&y) {
	T t = x;
	x = y;
	y = t;
}

template<typename T>
T max(T& x, T&y) {
	return x > y ? x : y;
}
```

Здесь мы говорим следующее: "Для любых произвольных типов **T** определена функция **swap** и **max**". 

Пример шаблонных классов (простейший это вектор):

```cpp
template <typename T>
class vector {
	T* arr;
	size_t sz;
	size_t cap;
};
```

Важно понимать, что при использовании шаблона для класса, генерируются разные классы, которые друг с другом не связаны и поэтому нельзя, например, приравнивать классы с разными шаблонами (но опять же можно это определить явно, автоматически не получится).

 В C++11 можно  делать шаблоны using-ов:
 
```cpp
template <typename T>
using mymap = std::map<T, T, T>
```

Кроме того, можно объявлять несколько мета-переменных типа через запятую:

```cpp
template <typename T, typename U>
void func(T x, U y) {
	// some code
}
```

# Использование
Для использования шаблонных функций не требуется явное указание типа, компилятор сам подставит нужный тип в функцию, т.к. это доступно на этапе компиляции.
Но если шаблон содержал только один тип, а подали 2 разных, то это ошибка компиляции, неявных преобразований в таком случае не производится, т.к. ищется шаблон.

```cpp
template<typename T>
void swap(T& x, T&y) {
	T t = x;
	x = y;
	y = t;
}

int main() {
	int x;
	int y;
	long z;
	
	swap(x, y); // OK
	swap(x, z); // CE
}
```

Но, если нужно, можно явно указать какой шаблон использовать:
```cpp
template<typename T>
void swap(T& x, T&y) {
	T t = x;
	x = y;
	y = t;
}

int main() {
	int x = 10;
	int y = 20;
	long z = 12l;
	
	swap(x, y); // OK
	swap<long>(x, z); // OK
}
```

И теперь ошибки для вызова нет, но это уже не касается шаблонов (проблема в конвертации типов ссылки, если бы передавали значение, то ошибки бы не было). Здесь явное указание шаблона и только теперь может быть неявное преобразование типов переменных, т.к. код уже сгенерирован.

Также доступно указание типа шаблона по умолчанию:

```cpp
#include <iostream>

template <typename T, typename U = int>
U f(T x) {
	std::cout << '1';
}

int main() {
	int x = 0;
	f<int>(x);
}
```

Для шаблонов с несколькими типами такие же правила передачи аргументов, как и для функций.
Но есть пример, где можно указывать не все типы:

```cpp
#include <iostream>

template <typename U, typename T>
U f(T x) {
	std::cout << '1';
}

int main() {
	int x = 0;
	f<int>(x);
}
```

Здесь достаточно задать явно только один тип, т.к. тип переменной **x** компилятор в состоянии определить самостоятельно. Но вот определить возвращаемое значение автоматически он не может.
# Что под капотом
По сути своей шаблоны это кодогенерация. На стадии компиляции, сразу после препроцессинга, компилятор генерирует код для вызванных шаблонов функций и по факту мы получаем разные функции с разными адресами в памяти и разный код. Причем код генерируется на основе вызова функции.
Сначала вызывается перегрузка шаблонов, потом инстанцирование, и потом уже специализация. 

# Перегрузка шаблонных функций

```cpp
#include <iostream>

template <typename T>
void f(T x) {
	std::cout << '1';
}

void f(int x) {
	std::cout << '2';
}

int main() {
	int x = 0;
	f(x);
}
```

Здесь нет ошибки компиляции. В данном случае выведет **2**. И это логично, общий случай работает для всех, но частный случай должен быть в приоритете.

```cpp
#include <iostream>

template <typename T>
void f(T x) {
	std::cout << '1';
}

void f(long long x) {
	std::cout << '2';
}

int main() {
	int x = 0;
	f(x);
}
```

Здесь наоборот выведется **1**, т.к. компилятор будет генерировать код для идеального соответствия типов (избегает конверсий).

Но если явно указывать, что генерировать надо с конкретным типом, то это обязывает компилятор сгенерировать код ибо выбора уже нет и это не ошибка компиляции.

```cpp
#include <iostream>

template <typename T>
void f(T x) {
	std::cout << '1';
}

void f(int x) {
	std::cout << '2';
}

int main() {
	int x = 0;
	f<int>(x); // 1
}
```


**ВАЖНО!**
При перегрузке шаблонных функций выбирается более частная.
Например:

```cpp
#include <iostream>

template <typename T, typename U>
void f(T x, U y) {
	std::cout << '1';
}

template <typename T>
void f(T x, T y) {
	std::cout << '2';
}

int main() {
	int x = 0;
	f(x, x); // 2
}
```
# Специализация шаблонов
Это нужно, когда требуется реализация отличная от общего представления. Например `vector<bool>`, где каждое значение в массиве меняет бит в памяти.
Это делается так:

```cpp
template <typename T>
class vector {
	T* arr;
	size_t sz;
	size_t cap;
};

// указываем, что не нужно дополнительно определять тип
template <>
// указываем явно какие аргументы принимает класс
class vector<bool> {
	char* arr;
	size_t sz;
	size_t cap;
};
```

Специализация также может быть частичной:

```cpp
template <typename T, typename U>
struct S {};

// Частичная специализация
// В скобках указываем, что все еще надо определить
template <typename W>
struct S<int, W> {};

template <typename W>
struct S<W, W> {};
```

Также частичная специализация может быть по ссылкам:

```cpp
template <typename T>
struct S {};

template <typename T>
struct S<T&> {};
```

В этом случае будут сгенерированы разные реализации для объектов с ссылочным типом и обычной переменной.
Но опять же, специализации выбираются по правилу чтения полного типа переменных.

Частичная специализация доступна только классам, для функций существует перегрузка и частичная специализация лишена смысла.

Для специализаций важен порядок, например:

```cpp
#include <iostream>

template <typename T, typename U>
void f(T, U) {
	std::cout << 1;
}

template <>
void f(int, int) {
	std::cout << 3;
}

template <typename T>
void f(T, T) {
	std::cout << 2;
}

int main() {
	f(0, 0); // 2
}
```

Здесь выведется **2**, поскольку специализация **(int, int)** указана для первого шаблона (в такой расстановке) и компилятор выберет шаблон с одинаковым типом, т.к. он наиболее подходящий. А в таком случае:

```cpp
#include <iostream>

template <typename T, typename U>
void f(T, U) {
	std::cout << 1;
}

template <typename T>
void f(T, T) {
	std::cout << 2;
}

template <>
void f(int, int) {
	std::cout << 3;
}

int main() {
	f(0, 0); // 3
}
```

Вывод будет **3**, т.к. это специализация шаблона с одинаковыми типами, который является специализацией шаблона для разных типов. Т.е. правило такое: выбирается наиболее частная подходящая специализация для вызова.

ОБЩЕЕ ПРАВИЛО ВЫБОРА ШАБЛОНА:
1) Сначала перегрузка между шаблонами (функции и шаблон).
2) Потом в выбранный шаблон подставляются типы и генерируются нужные версии.
3) Потом перегрузка из того, что получилось.

# Не типовые параметры шаблонов
Не только типы данных могут быть параметром шаблона. 
## Числовые
Так например в `std::array<int, 100>` создается массив из 100 целых чисел.
Как это написано:

```cpp
template <typename T, size_t N>
class array {
	T arr[N];
};
```

И это создаст на стеке массив с нужным размером.

Другой пример - матрицы:

```cpp
template <size_t M, size_t N, typename Field = double>
class Matrix {
	Field data[M][N];
};

template <size_t N, typename Field = double>
using SquareMatrix = Matrix<N, N, Field>;

int main() {
	// два разных класса
	Matrix<5, 5> m;
	SquareMatrix<5> sm;
}
```

Из такого объявления мы получили 2 различных класса, которые не связаны друг с другом и таким образом запретили конверсию между ними.
Но как теперь сделать умножение, ведь можно умножать квадратные матрицы на прямоугольные?

```cpp
template <size_t M, size_t K, size_t N, typename Field>
Matrix<M, N, Field> operator*(const Matrix<M, K, Field>& a, const Matrix<K, N, Field>& b);
```

Таким образом сразу же решается проблема с проверкой размерности матриц, это сразу будет CE, т.к. не будет существовать такого шаблона.
Но для шаблонных числовых параметров должны подаваться только константные значения, которые известны на этапе компиляции.

ВАЖНО !
Параметры шаблона принимают только целочисленные (интегральные) аргументы.
Но с C++20 можно и литералы тоже.

## Шаблонные параметры
Разберем на примере реализации stack
Допустим мы хотим иметь в качестве параметра контейнер, поверх которого будет оборачиваться стек (стек может быть реализован, как на деке, так и на списке).

Чтобы задать тип контейнера нужно написать так:

```cpp
template <typename T, template<typename> typename Container>
class Stack {
	Container<T> container;
};
```

Здесь вторым аргументом указано, что параметром стека является другой тип, который тоже имеет шаблонные параметры.
Его можно сделать по умолчанию:

```cpp
template <typename T, template<typename, typename> typename Container = std::vector>
class Stack {
	Container<T, std::allocator<T>> container;
};
```

Тип в векторе передавать не нужно, т.к. это будет объект, а нужно передать шаблон. `std::alllocator` это часть STL контейнера и пока знать о ней не нужно, но это является частью шаблона и при передаче шаблона нужно учитывать и аргументы по умолчанию.

Лучше написать так:
```cpp
template <typename T, template<typename> typename Container = std::vector>
class Stack {
	Container<T> container;
};

template <typename T>
using myvector = std::vector<T>;

int main() {
	// теперь компилируется и лучше работает с другими контейнерами
	Stack<int, myvector> s;
}
```

# Вычисления на шаблонах
Разберем на примере Фибоначчи

```cpp
template <int N>
struct Fibonacci {
	static const int value = Fibonacci<N-1>::value + Fibonacci<N-2>::value;
};
```

У шаблонов тоже есть рекурсия и она фиксированной глубины. И такой код не скомпилируется - будет вызвана ошибка глубины рекурсии шаблона. Значит требуется добавить специализацию для конечной рекурсии:

```cpp
template <int N>
struct Fibonacci {
	static const int value = Fibonacci<N-1>::value + Fibonacci<N-2>::value;
};

template <>
struct Fibonacci<0> {
	static const int value = 0;
};

template <>
struct Fibonacci<1> {
	static const int value = 1;
};
```

(Переменные все static т.к. объекты не создаются, а значение нужно сделать частью типа).
Такое вычисление работает за линейное время и генерирует 21 класс чисел Фибоначчи, что по сути своей мало отличимо от динамики.
