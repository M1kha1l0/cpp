# Работа с памятью
Для каждого процесса ОС выделяет определенный кусок виртуальной памяти, которая связывается с физической памятью с помощью механизмов memorymapping-а.
Такой подход обеспечивает: 
- безопасность программ (чтобы не поубивали друг друга какой-нибудь случайной записью в чужую память).
- виртуальная память представляет собой один непрерывный кусок памяти в то время, как физическая память поделена на куски.

## Разделы виртуализированной памяти процесса

## Data (static memory)
Область статической памяти процесса (размер известен при запуске).
В ней хранятся **глобальные переменные**, static переменные, объявленные литералы, **переменные объявленные внутри namespace**, а также некоторая другая информация (не код), которая живет на всем времени исполнения процесса.

Пример переменной выделенной на Data:

```cpp
#include <iostream>

void f() {
	static int x = 0;
	std::cout << ++x << '\n;
}
```

В примере переменная **x** выделяется на Data и имеет неизменный адрес в памяти на время жизни программы. Он инициализируется один раз при первом заходе в функцию, а далее просто берется его адрес, при этом все изменения сохраняются и далее переменная повторно не инциализируется, а просто берется значение по адресу.
## Text
 Раздел памяти виртуализированного пространства, в котором находится машинный код текущей программы (размер фиксирован и известен при запуске). Это сделано для того чтобы подгружать процессору то, что он должен  исполнять быстро.

## Stack (automatic memory)
Ограниченный раздел виртуальной памяти процесса (обычно 8Мб), где хранятся все созданные локальные переменные в коде (но не динамические). Оптимизируется компилятором и ручное взаимодействие со стеком памяти не нужно.
Стек заполняется последовательно и хранит в себе указатель на конец предыдущей занятой ячейки памяти.

```c
#include <stdio.h>

int main() {
	int array[10]; // локальный массив размера 10
	printf("size of array: %d\n", sizeof(array) / sizeof(int));
	return 0;
}
```

После завершения программы система сама освободит область памяти, где лежит **array**
Но если общий размер переменных или размер стека вызова больше размера стека, то программа завершается с ошибкой StackOverflow (переполнение стека вызова может быть вызвано бесконечной рекурсией, даже если вызывается без аргмуентов и локальных переменных, т.к. хранится адрес возврата).
### Стек вызова функций
На стеке также лежит стек вызова функций.
Когда происходит обращение к функции, кроме загрузки локальных переменных, еще нужно записать очень важную вещь - **адрес возврата**. Ибо мы знаем в сегменте Text адрес реализации функции, но не знаем куда переместиться после исполнения этого кода и для этого хранится адрес возврата, как дополнительный указатель (перед заходом в функцию сохраняется адрес возврата, а потом уже все что нужно в функции).

## Heap 
Куча - динамический раздел памяти процесса.  Требует ручного взаимодействия с памятью, т.е. выделять память под переменную и освобождать по окончанию программы, если память не освобождать, то **будет утечка памяти**. Куча не имеет ограничений по размеру и поэтому может разрастаться по мере необходимости.
Есть функции, которые позволяют управлять выделяемой памятью.
### malloc/calloc
**malloc** - функция для выделения памяти для объектов произвольного размера. Запрашивает у ОС раздел памяти равный количеству байт в аргументах.
malloc не гарантирует выделение памяти и в случае ошибки выделения вернет нулевой указатель (NULL). 
malloc возвращает byte-array и требует явного приведения типа к нужному указателю.
`void* malloc(size_t bytes_count);`

**calloc** - функция аналогичная malloc. Принимает на вход размер типа и количество элементов массива с таким типом. Возвращает указатель с заданным типом
`type* calloc(type, size_t bytes_count);`

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
	int size = 10;
	int *array1 = (int*)malloc(size * sizeof(int)); // return void*
	int *array2 = calloc(sizeof(int), size); // return int*
	
	free(array1);
	free(array2);
}
```

Удалить динамический массив выделенный с помощью **malloc** позволяет функция **free**.
Важно учесть, что **free** вызывает ошибку сегментации, если память дважды освобождается. Это объясняется тем, что когда мы освобождаем память в первый раз, мы больше не владеем участком этой памяти и она может быть уже занята другим, а когда вызываем **free** по указателю освобожденной памяти, то происходит запрос к операционной системе на освобождение, а там уже говорят, что память вам не принадлежит и выходит с ошибкой.
Также разыменования освобожденного указателя приводит к UB, т.к. память уже не во владении программы.
Также **free** очень чувствителен к указателю, по которому очищается память. Любые изменения относительно выделенного указателя на память это будет UB.
### new[]

Является оператором, доступным в C++.
Это аналог malloc в C, возвращает сразу указатель нужного типа. Запрашивается `sizeof(Type)` умноженное на число переданное в аргумент.

Пример использования для выделения динамической памяти на 10 элементов:
```cpp
#include <iostream>

int main() {
	int size = 10;
	int *array = new int[size];
	
	return 0;
}
```

 Важно заметить, что есть 2 варианта выделения памяти `int()` и `int[]`. Первое выделит указатель на один элемент с значением в скобках, а второе выделит массив размера указанного в скобках.
 Можно провернуть один трюк, который в чистом виде является UB
 
 ```cpp
 #include <iostream>

int main() {
	int size = 10;
	int *array = new int[size];
	
	std::cout << "size of array: " << array[-2] << '\n';
	
	delete[] array;
	// delete array - неправильно, UB
}
 ```

Под капотом **new** выполняет множество низкоуровневых операций и кроме выделенного массива хранит еще какую-то информацию для корректного удаления.

Из-за такой особенности работы оператора **new** для разных типов выделенной памяти разный оператор удаления.
Так например **delete[]** - удаляет массив по указателю, а **delete** удаляет только указатель. Неправильное использование этих операторов считается UB и предсказать, что конкретно произойдет очень сложно.
Также нельзя освобождать с помощью **delete** память, которую выделили через **malloc**, из-за разного процесса удаления это вызовет UB (Наоборот тоже нельзя).
Аналогично **free**, **delete** вызывает ошибку сегментации, если дважды освободить память, а также, если изменить выделенный указатель.
