# Типы данных и их операции

## Типизация
1) **Статическая** - типы данных всех переменных известны на этапе компиляции и неизменяемы в рантайме.
2) **Динамическая** - в данном случае типы данных меняются в рантайме.

У C++ статическая типизация переменных.

## Типы данных

### Целочисленные типы данных

| Type      |        Signed         | Unsigned | Вес в байтах |
| --------- | :-------------------: | :------: | :----------: |
| int       | $[-2^{31}; 2^{31}-1]$ |    $$    |      4       |
| long      |                       |          |      4       |
| long long |                       |          |      8       |
| short     |                       |          |      2       |
| char      |                       |          |      1       |
Для всех типов данных не гарантируется сколько они занимают места в байтах. На разных системах может занимать разное число байт.

Но есть аналог для данных переменных: **intXX_t** и **uintXX_t**
Эти типы данных фиксированы по ширине и описаны в стандарте. На месте **XX** - число бит по ширине.
Например тот же int будет  иметь запись int32_t, а его беззнаковая версия uint32_t.
Это касается всех целочисленных типов данных.

```C++
// не всегда 4 байта
int x;
unsigned int y;

// всегда 4 байта не зависимо от системы
int32_t z;
uint32_t p;
```

#### Представление в памяти
Берется непрерывный кусок байт и в нем в двоичной записи записывается число. Но представление знаковых и безнаковых отличается. У знаковых первый бит резервируется под знак, где 0  - больше 0, 1 - меньше 0.
Подробнее позже.

#### size_t
Тип данных по свойствам равный **unsigned long long**. Используется для индексации ячеек в памяти.
#### std::byte
Тип данных введенный для работы со сложением переменных типа **char**.

#### bool
**bool** - логический тип данных неявно приводимый из **int**. Значение **true** присваивается если записанное число в него не равно 0, **false** если 0.
Занимает 1 байт, не бит. Это обусловлено устройством адресации в памяти - нельзя присвоить целочисленных адрес переменной весом в 1 бит. Иначе, если в памяти 2 bool подряд, то неизвестно как их отличить.

Для компактности хранения **bool** есть std::vector, который преобразует 8 переменных типа bool в один байт.

### Типы с плавающей точкой

| Тип         | Singned | Unsigned | Вес в байтах |
| ----------- | ------- | -------- | :----------: |
| float       |         |          |      4       |
| double      |         |          |      8       |
| long double |         |          |      16      |
Так же как и с целыми числами не гарантируется для каждой системы строгий вес.
Гарантируется только, что int влезает полностью в double без потери точности (не наоборот).
#### Представление в памяти
Знаковое представление как у целых чисел.
Существует 2 раздела блока памяти: **мантиса** и **экспонента**.

Мантиса представляет собой численное преставление.
Экспонента же указывает в какую степень двойки (положительную или отрицательную) нужно возвести число.

### Литералы

```C++
// числовые литералы
int x = 5;
char c = 'a';
double d = 2.3;
bool b = true;

// строковый литерал
char s[5] = "1234"; // const char*
```

Каждый литерал захардкожен для каждого типа. По умолчанию просто число это **int**, если число с точкой - **double**. С символами и строками понятно.

#### Целочисленные литералы

Для целочисленных литералов есть записи в 4 системах исчисления:  8, 10, 16.
Записываются они так:
```C++
int a = 0xFF; // 256 в 16-ой
int b = 0400; // 256 d 8-ой
```

Получаем, что литералы с 0 это число в восьмеричной системе, а 0х это в шестнадцатеричной.

#### Литеральные суффиксы

Литеральные суффиксы - определенная конструкция для изменения типа литерала.

```C++
uint32_t = 5u;  // unsigned 5
float f = 0.2f; // 0.2 в float т.к. изначально double
```

Суффиксы литералов нужны, если есть 2 схожих типа по диапазону, но обрабатываются они по-разному.

```C++
#include <iostream>

void foo(uint32_t x) {
	std::cout << "unsigned int\n";
}

void foo(int32_t x) {
	std::cout << "signed int\n";
}

int main() {
	foo(5); // выведет signed int
	foo(5u); // выведет unsigned int
	return 0;
}

```


## Операции
Поддерживаются все арифметические операции для всех базовых типов, а также операторы сравнения.
### Преобразования

Основное правило безопасного преобразования типов - это приведение числа к типу с большем диапазоном значений как для целых чисел, так и для чисел с плавающей точкой. Называется это **Integer/Floating promotions**.
Преобразование в обратную сторону небезопасно из-за потери точности

```C++
short x = 5;
int y = x; // безопасно
short z = y; // небезопасно
```

Также возможно приведение чисел с плавающей точкой к целым числам и наоборот, если конечное число полностью покрывает диапазон приводимого.
НО безопасное приведение только из целого в число с плавающей точкой, иначе при приведении к целому дробная часть отбрасывается и теряется точность.

Также преобразования делятся на 2 класса:
#### Явные
Явные преобразования делаются непосредственно пользователем. Например с уже рассмотренными суффиксными литералами.

```C++
uint32_t x = 5u;
```

#### Неявные
Неявные преобразования типов происходят в основном в арифметических операциях по правилам преобразования числа к типу с большим диапазоном (Integer/Floating promotions).

```C++
int x = 5;
long long y = 10;

auto z = x+y; // будет long long
```

Это касается не только арифметики, но и сравнения, а также приведение к bool.

Но неявные преобразования не очень безопасны:
```C++
auto x = 10 + 1u; // x - unsigned int
auto y = 0 - 1u;  // это 4 милларда, из-за преобразования в unsigned int
```

#### Конверсия и Промоушен
Конверсией называется конвертацией в типы с потерей точности.
Промоушеном называется приведение к типу без потери точности.
### std::cin и std::cout
На самом деле при попытке вывести что-то в консоль вызывается оператор '<<' или '>>' класса **ostream** **или** **istream** соответственно. Операторы для разных типов каким-то правильным образом определены для передаваемого параметра.

```C++
int x;
std::cout << x; // в данном случае x - передаваемый параметр
```

### Операторы
Все основные операторы: +, -, /, умножение, %, <<, >>, >=, <=, = (двойное), ||, &&, <, >, |, &, ^.
Определены только для целых: %, >>, <<, |, &, ^.
Логические: &&, ||, >=, <=, >, <, =, !=

#### Ленивые операции
Для операторов && и || программа не будет вычислять правое значение функции, если левое значение определяет истинность выражения.

```C++
// если vector.size() < 5, то поведение программы
// не изменится, если вторая часть падает с ошибкой
// будет значение false
if(vector.size() >= 5 && v[4] == 1) {
}

```

Пример аналогичен для ||

#### Операторы присваивания
Всегда присваивается левому значению даже если это выполняется несколько раз

```C++
#include <iostream>

int main() {
	int x = 10;
	int y = 20;
	int z = 30;
	std::cout << (x = y) << '\n'; // 20
	std::cout << (x = y) = z << '\n' // 30
}
```

По сути после каждого присваивания возвращается ссылка на левую переменную и ее уже дальше можно присваивать. Т.е. после операции присваивания возвращается ссылка (в примере) на **x**, это **lvalue**. На пальцах это тип выражения, после которого значению еще что-то можно присвоить. Например все унарные операторы **lvalue**, все операторы для двух значений - **rvalue** (нельзя присвоить a+b какое-то значение, т.к. оно временное).
