## Псевдонимы 
### Reference (lvalue reference)
Ссылка. Ссылка является псевдонимом объекта. Обязательно должна быть инициализирована.

```cpp
#include <iostream>

int main() {
	int x = 10;
	int &y = x;
	std::cout << x << " " << y << '\n'; // "10 10"
	y = 25;
	std::cout << x << " " << y << '\n'; // "25 25"
	x = 23;
	std::cout << x << " " << y << '\n'; // "23 23"
	return 0;
}
```

UB:
```cpp
#include <iostream>

int& foo() {
	int i = 10;
	return i;
}

int main() {
	int& i = foo();
	std::cout << i; // undefinded behaivour
	return 0;
}
```

Поскольку ссылка определена на стеке, то после завершения `foo()` место освободится, а значит ссылка никуда не ведет и получается UB.

`const Type&` - делает **lvalue** ссылку доступной для конвертации в **rvalue** ссылку.

## Перегрузка функций

Несколько функций имеют одинаковые названия , НО разные аргументы. При этом  передаваемые аргументы в функцию должны быть явно указаны, иначе компилятор не будет знать к какому типу преобразовывать переменную.

```C++
int foo(int x) {}

int foo(float x) {}

int main() {
	foo(0.0d); // ошибка компиляции
}
```

В данном примере мы пытаемся передать в функцию аргумент, который должен конвертироваться в тип с меньшей точностью. Для компилятора это равнозначные функции, т.к. приводится с потерей точности, а значит неоднозначное определение.
Компилятор считает, что функция определена однозначно, если аргументы приводятся промоушеном.

Еще один пример с перегрузкой. Нельзя перегружать функцию разным возвращаемым значением, но с одинаковыми аргументами.

```C++
// ошибка компиляции
int foo() {}
void foo() {}

// корректное определение
int foo(int x) {}
void foo() {}
```

Также нельзя перегружать функции ссылками, т.к. они неотличимы от обычной переменной:

```cpp
void foo(int& x) {}
void foo(int x) {}

int main() {
	int x = 10;
	foo(x); // CE
}
```

Но:
```cpp
void foo(int& x) {}
void foo(int x) {}

int main() {
	foo(1); // OK
}
```

Т.к. первая функция принимает **lvalue**, а вторая **rvalue**
#### Variadic functions
Функции, которые принимают сколько угодно аргументов. Унаследовано из C, например, для таких функций как  **printf** и производных. Такая функция имеет обязательные аргументы (идут в начале) и многоточием обозначается после какого аргумента принимаются все остальные.

```cpp
void simple_print(const char* fmt...) {
	// some code
}

simple_print("abcd");
simple_print("abcd", 1, '1', 12);
simple_print("abcd", 1, "1234", '1', 12);
```

Эта функция обязательно принимает какую-то строку, а далее любые аргументы.
Реализация функции основана на макросах в C.
## Пространства имен

Пространства имен позволяют ограничить видимость объявленных сущностей в глобальном скоупе. Это полезно, когда проект огромный и названия могут пересекаться, тогда обращение к разным namespace-ам дает результат, который объявлен в них.
Namespace-ы не могут, как другие сущности, быть просто объявлены, их нужно  определять сразу после объявления.

```C++
#include <iostream>

namespace X {
	int x = 10;
}

namespace Y {
	int x = 12;
}

int main() {
	int x = 20;
	std::cout << x << '\n';    // 20
	std::cout << X::x << '\n'; // 10
	std::cout << Y::x << '\n'; // 12
}
```

Существует также 2 вида обращения к сущностям:
- Квалифицированный идентификатор. Пример:  `x;`.
- Неквалифицированный идентификатор. Пример:  `namsp::x;`.
### using
Позволяет расширить область видимости данного скоупа. При использовании **using**-а к объявленным сущностям в данном скоупе добавляется конкретные или все сущности из пространства имен.

#### Обращение с переменными
```C++
#include <iostream>

namespace X {
	int x = 10;
}

namespace Y {
	int y = 20;
}

int main() {
	std::cout << x << '\n'; // Ошибка! х не объявлен
	using X::x; // Вытаскиваем объявленный х из протранства имен
	std::cout << x << '\n'; // 10
}
```

Расширение области видимости доступно и для глобального скоупа. В таком случае они будут видны как глобально объявленные сущности.

```C++
#include <iostream>

namespace X {
	int x = 10;
}

namespace Y {
	int x = 20;
}

using X::x; // теперь глобальная переменная

int main() {
	std::cout << x << '\n';    // 10
	std::cout << Y::x << '\n'; // 20
}
```

#### Обращения с пространствами имен
Кроме того можно расширять область видимости пространств имен.

```C++
#include <iostream>

namespace X {
	int x = 10;
}

namespace Y {
	using X::x;
	int y = 20;
}

int main() {
	std::cout << Y::x << '\n'; // 10
}
```

Если же нужно объявить все, что находится в пространстве имен

```C++
#include <iostream>

namespace X {
	int x = 10;
}

namespace Y {
	using X::x;
	int y = 20;
}

using namespace Y; // объявленно все из Y

int main() {
	std::cout << x << '\n'; // 10
}
```

НО глобальное объявление очень плохая практика, т.к. глобальный **using** объявляет все внутренние функции и в коде юзера может  быть функция с такой же сигнатурой. В таком случае будет перегрузка функции и узнать, что именно вызывается (функция из namespace или функция юзера) будет возможно только при запуске, компилятор подскажет только с неоднозначным объявлением перменных.

#### Обращение с типами
Using позволяет определить пользовательские типы данных.

```C++
#include <vector>

int main() {
	using vi = std::vector<int>;
	vi v; // = std::vector<int> v
	
	using ll = long long;
	ll l; // = long long l
}
```

#### Обращение к глобальному namespace

```C++
#include <iostream>

int x = 10;

namespace N {
	int x = ::x; // обращение к глобальному пространству имен
}

int main() {
	std::cout << N::x << '\n'; // 10
}
```
