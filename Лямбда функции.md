Базовое использование лямбд - хотим сортировать, например, вектор по некоторому условию:

```cpp
#include <iostream>
#include <algorithm>
#include <vector>

int main() {
	std::vector v = {5, -3, 2, 4, -1};
	std::sort(v.begin(), v.end(), 
			  [](int x, int y){ return x*x < y*y; });
			  
	for(const auto& i : v) {
		std::cout << i << ' ';
	}
}
```

Выведется массив сортированный по возрастанию квадрата.

Лямбду, которую передали в аргументы, можно и сохранить в переменную и также ее передать:

```cpp
#include <iostream>
#include <algorithm>
#include <vector>

int main() {
	std::vector v = {5, -3, 2, 4, -1};
	auto cmp = [](int x, int y){ return x*x < y*y; };
	std::sort(v.begin(), v.end(), cmp);
			  
	for(const auto& i : v) {
		std::cout << i << ' ';
	}
}
```

И переменная cmp имеет тип этой лямбды. И причем нет конкретного имени, компилятор самостоятельно генерирует для него имя и в логах об ошибке, например, если что-то не так с лямбдой, выведется `main()::<lambda(args...)>`. Также имя лямбды возможно узнать из `typeid`.
По сути своей лямбда это функциональный объект для которого определен оператор ().

Также для лямбд возможна передача типа в шаблонные аргументы:

```cpp
#incluse <iostream>
#include <set>

int main() {
	auto cmp = [](int x, int y){ return x*x < y*y; };
	// передаем тип компаратора и его значение
	std::set<int, decltype(cmp)> s(cmp);
	
	s.insert(-1);
	s.insert(-3);
	s.insert(2);
	
	for(const auto& i : s) {
		std::cout << i << ' ';
	}
}
```

Порядок элементов сета будет определен переданной лямбдой.
До C++20 передача компаратора по значению была обязательной, сейчас же это необязательно. Строка объявления выглядит короче:
`std::set<int, decltype(cmp)> s;`
Это было из-за отсутствия конструктора по умолчанию в ранних версиях.

Есть важный нюанс касательно определения лямбд:

```cpp
int main() {
	auto f = [](int x, int y){ return x*x < y*y; };
	assert(f == [](int x, int y){ return x*x < y*y; });
}
```

Даже если лямбды одинаковые по сигнатуре, компилятор генерирует для каждой уникальное имя.

Для того чтобы понять, какой тип возвращает лямбда функция, введем понятие **Closure** (замыкание). Это типы которые возвращает лямбда генерируемые компилятором.
Т.е. сама по себе лямбда это выражение, а возвращаемый тип это замыкание. Объекту, которому присваивается лямбда, называется объектом замыкания.
Компилятор определяет возвращаемый тип также, как и в обычной функции с auto, правила вывода типов одинаковые.
Но можно явно указать какой тип возвращает лямбда, с помощью стрелки:
`[](int x, int y) -> bool { return x*x < y*y; }`
Это полезно когда нужно возвращать ссылки, на месте bool может стоять decltype(auto)

Кроме того, лямбды можно вызывать на месте, при их создании. Достаточно после выражения вызвать оператор () с необходимыми аргументами. Это нужно, например, когда для определения константного объекта требуется сложная логика, например:

```cpp
int main() {
	const int x = [](int y){
		if(y % 2 == 0) {
			return x / 2;
		}
		return 3 * y + 1;
	}(2024);
}
```

Кроме того такой случай корректен:

```cpp
int main() {
	[]()[]();
}
```

Присваивание лямбд доступно только, если лямбда не захватывает никаких значений.

## Захват значений

Это необходимо, когда нужно выполнять какое-то действие относительно одного постоянного параметра, который находится в скоупе с этой лямбдой (локальная переменная) и при этом его нельзя передать, как дополнительный параметр.
Например, сортировка по расстоянию до какой-то одной точки:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
	std::vector v = {1, 2, -3, 4, -5, 6};
	int c = 3;
	
	auto cmp = [](int x, int y){
		return (x-c)*(x-c) < (y-c)*(y-c);
	};
	
	std::sort(v.begin(), v.end(), cmp);
}
```

Но это не валидный код, в скоупе cmp не объявлена переменная **c**. Можно использовать глобальную переменную, но это не всегда полезно. Чтобы объявить, то к чему должен быть доступ, но нет в параметрах лямбды, нужно написать имя переменной в квадратных скобках:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
	std::vector v = {1, 2, -3, 4, -5, 6};
	int c = 3;
	
	auto cmp = [c](int x, int y){
		return (x-c)*(x-c) < (y-c)*(y-c);
	};
	
	std::sort(v.begin(), v.end(), cmp);
}
```

Это заработает, как если бы в структуре появилось поле с переданной переменной.

Но здесь кроется коварный нюанс, если передать в список захвата только имя переменной, то оно копируется! Чтобы передавать их по ссылке нужно перед именем написать амперсанд:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
	std::vector v = {1, 2, -3, 4, -5, 6};
	int c = 3;
	
	auto cmp = [&c](int x, int y){
		return (x-c)*(x-c) < (y-c)*(y-c);
	};
	++c;
	
	// т.к. передали по ссылке, порядок тоже изменится вместе с переменной
	std::sort(v.begin(), v.end(), cmp);
}
```

Но вопрос, какой модификатор константности ссылки у захваченной переменной? В варианте выше это захват по не константной ссылке.
Но если мы принимали по значению, то там какой модификатор константности? Все типы переданные по значению в лямбду константные.
Кажется нелогичным. Это связано с тем, что оператор () константный метод, и на захваченные переменные по значению навешивается const справа и на поле ссылки это не влияет вообще, а на поле значения еще как.

Теперь задача стоит такая: как сделать так, чтобы переменные переданные по значению изменялись? Чтобы отменить константность метода используют **mutable** для метода:

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
	std::vector v = {1, 2, -3, 4, -5, 6};
	int c = 3;
	
	// теперь можем менять c внутри лямбды
	auto cmp = [c](int x, int y) mutable {
		++c;
		return (x-c)*(x-c) < (y-c)*(y-c);
	};
	
	std::sort(v.begin(), v.end(), cmp);
}
```

При этом значение переданное в лямбду будет меняться при каждом ее  вызове, т.к. это по сути поле внутри структуры.

Но нужно учитывать, что некоторые методы и контейнеры ожидают, что лямбда будет иметь константный оператор. Логи компилятора для такой ошибки совсем уж непонятные. Например, std::set ожидает константный компаратор, кроме того, когда добавляются захваченные значения, то удаляется конструктор по умолчанию, поэтому лямбду еще надо будет передать в конструктор объекта.

Кроме того, доступен захват с инициализацей:

```cpp
int main() {
	int c = 1;
	auto f = [c = c + 1](){};
}
```

Здесь первая c это уже поле структуры, а следующая это переменная из текущего скоупа. это подталкивает на мысль, что в списке инициализации может быть выражение, но нет, можно захватить только по ссылке или по значению с или без инициализации.

Но еще хочется, чтобы захваченный объект нельзя было поменять, но при этом получить по ссылке. Захват по константной ссылке выглядит так:

```cpp
int main() {
	int c = 1;
	auto f = [&c = std::as_const(c)](){};
}
```

Важно, что без амперсанда не сработает, т.к. это будет захват по значению.

Чтобы захватить несколько значений, нужно  просто перечислить их через запятую.
С этим появилась возможность захватывать абсолютно все из области видимости (все локальные переменные) с помощью синтаксической конструкции: `[=]` - захватить все по значению, и `[&]` - захватить все по ссылке.
Так делать не рекомендуется, позже будет видно почему.

также можно захватывать пакеты:

```cpp
template <typename... Args>
void test(Args... args) {
	auto f = [&args...](){};
}
```

Здесь все значения в аргументах приняты по ссылке. Подробнее посмотреть как принимать пакеты в список захвата можно [здесь](https://en.cppreference.com/w/cpp/language/lambda.html).

Еще лямбда может захватывать свой объект:

```cpp
#include <iostream>

struct S {
	int a = 0;
	auto get_lambda() {
		auto f = [=](int x) {
			return x + a;
		}
		return f;
	}
};

int main() {
	S s;
	auto f = s.get_lambda();
	
	std::cout << f(5);
}
```

Здесь все хорошо вывод будет ожидаемый (8).
Но все меняется, когда объект внезапно удаляется:

```cpp
#include <iostream>

struct S {
	int a = 0;
	auto get_lambda() {
		auto f = [=](int x) {
			return x + a;
		}
		return f;
	}
};

int main() {
	S* s = new S();
	auto f = s->get_lambda();
	delete s;
	std::cout << f(5);
}
```

Здесь уже UB. Как обсуждали раннее лямбда с таким захватом заберет только все локальные переменные, поле **а** здесь не локальное.
Как такового захвата полей не происходит, если захват всего делается внутри класса. На самом деле захватывается **this** и уже из него лямбда получает доступ к полям класса.
Но сказать лямбде, что нужно захватывать именно поля класса можно явно:

```cpp
#include <iostream>

struct S {
	int a = 0;
	auto get_lambda() {
		auto f = [this](int x) {
			return x + a;
		}
		return f;
	}
};

int main() {
	S* s = new S();
	auto f = s->get_lambda();
	delete s;
	std::cout << f(5);
}
```

Но это все еще не решает проблему, захват полей все еще происходит по указателю.
Т.к. поле класса не является локальной переменной, то в список захвата просто написать ее имя будет ошибкой, но если воспользоваться захватом с инициализацией, то все будет работать корректно:

```cpp
#include <iostream>

struct S {
	int a = 0;
	auto get_lambda() {
		auto f = [a = a](int x) {
			return x + a;
		}
		return f;
	}
};

int main() {
	S* s = new S();
	auto f = s->get_lambda();
	delete s;
	std::cout << f(5);
}
```

Захват static переменных.
Что произойдет в таком коде:

```cpp
#include <iostream>

int main() {
	static int x = 0;
	
	auto f = [](=){++x;};
	
	f();
	std::cout << x;
}
```

Будет вывод 1. Кроме того, если это статическая переменная, то захват можно не писать, она и так видна из лямбды (они все общие): `auto f = [](){++x;};`
Отсюда вывод: статическая переменная не захватываемая, компилятор просто проигнорирует ее захват, но выведет предупреждение.

# Внутренности лямбды
На сайте [cppinsights.io](https://cppinsights.io/)  можно посмотреть какой на самом деле класс генерирует компилятор, когда объявляется лямбда.

Среди прочего и без того очевидного, есть определенный оператор приведения лямбды к указателю на функцию. Таким образом лямбда может передаваться куда-то где требуется указатель на функцию.
Но и это не все, такой каст должен возвращать указатель на функцию и для этого создается статическая и приватная функция `__invoke`, которая возвращает результат явного вызова оператора ().

Если в лямбде есть захват, то добавляется приватное поле с типом захвата и именем переменной захвата. Заметим, что тогда указатель на функцию создать невозможно, непонятно каким образом это будет работать.
Если захватывается все, то компилятор оптимизирует это до используемых переменных, лишние данные в классе не хранятся.

# От лямбды к C-функции
В синтаксисе лямбд поддерживается преобразование к C-функции явно:

```cpp
auto f = +[](int x, int y){ return x < y; };
```

Здесь в f будет лежать именно указатель на C-функцию.

# Обобщенные лямбды
По сути своей это шаблонные лямбды, где принимаемые значения имеют тип auto.

```cpp
#include <iostream>
#include <map>
#include <algorithm>

int main() {
	std::map<std::string, int> m;
	m["abc"] = 1;
	m["efg"] = 2;
	
	decltype(m) m2;
	auto f = [](auto& item){
				++item.second;
				return item;
			};
	
	std::transform(m.begin(), m.end()
		std::inserter(m2, m2.begin()),
		f
	);
}
```

В замыкании f принципиально меняется один момент - не возвращаемый класс становится шаблонным, а оператор ().

Ну раз лямбды бывают шаблонными, то мы можем добавить requires и концепты для лямбд после добавления аргументов:

```cpp
#include <iostream>
#include <map>
#include <algorithm>

int main() {
	std::map<std::string, int> m;
	m["abc"] = 1;
	m["efg"] = 2;
	
	decltype(m) m2;
	auto f = [](auto& item)
			requires requires(decltype(item) i) { i.second; }
			{
				++item.second;
				return item;
			};
	
	std::transform(m.begin(), m.end()
		std::inserter(m2, m2.begin()),
		f
	);
}
```

Также перед указанием аргументов можно явно указать принимаемый шаблон функции и тогда выйдет чуть красивее:

```cpp
#include <iostream>
#include <map>
#include <algorithm>

int main() {
	std::map<std::string, int> m;
	m["abc"] = 1;
	m["efg"] = 2;
	
	decltype(m) m2;
	auto f = []<typename T>(T& item)
			requires requires(T i) { i.second; }
			{
				++item.second;
				return item;
			};
	
	std::transform(m.begin(), m.end()
		std::inserter(m2, m2.begin()),
		f
	);
}
```

Но это были requires для оператора (), но можно еще указать requires для типов. Это делается перед объявлением аргументов.

## Функциональное программирование
Это парадигма, в которой функции порождают другие функции. И очень удобно, что у нас есть обобщенные лямбды. Посмотрим как из одних лямбд можно создавать другие.
Мы хотим чтобы данный код компилировался:

```cpp
int main() {
	auto sum = [](int x, int y) { return x + y; };
	// implement
	auto sum_with_five = fix_second_argument(sum, 5);
	
	assert(sum_with_five(3) == 8);
}
```

Первое что приходит в голову: нужно хранить переданную лямбду и аргумент. И это является решением:

```cpp
int main() {
	auto sum = [](int x, int y) { return x + y; };
	auto fix_second_argument = [](auto&& f, auto&& second_arg) {
		return  [f = std::forward<decltype(f)>(f),
					second_arg = std::forward<decltype(second_arg)>(second_arg)]
				(auto&& first_arg)  {
					return f(std::forward<decltype(first_arg)>(first_arg),
						   std::forward<decltype(second_arg)>(second_arg));
				}
	};
	auto sum_with_five = fix_second_argument(sum, 5);
	
	assert(sum_with_five(3) == 8);
}
```

Заметим, что мы захватываем необходимые параметры для выходной лямбды.
Как это работает?

> "Ехал лямбда через лямбду, видит лямбда в лямбде лямбда, сунул лямбда в лямбду лямбду, лямбда лямбда лямбда лямбда".
> - Мещерин Илья

Еще пример. Будем делать функцию с двойным выполнением:

```cpp
#include <iostream>

int main() {
	auto print_hello = [](){ std::cout << "Hello!\n"; };
	
	auto do_twice = [](auto&& f){
		return [f = std::forward<decltype(f)>(f)](){ f(); f(); };
	};
	
	auto print_hello_hello = do_twice(print_hello);
	
	print_hello_hello();
}
```

# Рекурсивные лямбды
Разберем на примере подсчета чисел Фибоначчи:

```cpp
int main() {
	auto fibonacci = [](int n){
		return n > 2 ? fibonacci(n-1) + fibonacci(n-2) : 1;
	};
}
```

Наивная реализация и не работает. Компилятор не найдет определения лямбды внутри себя.

Но есть решение: [[Вывод типов|deducing this]] (неявно принять себя):

```cpp
int main() {
	auto fibonacci = [](this auto& self, int n) -> int {
		return n > 2 ? self(n-1) + self(n-2) : 1;
	};
}
```

Но здесь нужно обязательно указать возвращаемый тип, потому что в шаблонных подстановках запрещена рекурсия. А рекурсивное определение типа происходит из-за вызова self.
