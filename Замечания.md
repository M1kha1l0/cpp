`if constexpr` - на самом деле это синтаксический сахар. Для написания функций ранее (ранее C++17) использовали перегрузки функций.  Появлялась новая функция `function_helper`, которая имела дополнительный аргумент - условие `if constexpr` и по переданному условию по шаблонам инстанцировалось в нужную реализацию.

---
Вставка в вектор своего же элемента:

```cpp
#include <vector>
#include <string>

int main() {
	std::vector<std::string> v(5, "abc");
	v.push_back(v[3]);
}
```

Проблема в том, что при реаллокации, все элементы перемещаются в другой массив данных и обращение по v[3] это битая ссылка, объект по этой ссылке уже перемещен и уничтожен. Поэтому в векторе это сделано так: сначала объект вставляем, а потом уже реаллоцируем.

---
В современном коде принято принимать по значению объекты, если заведомо известно, что они все перемещаемые. В таком случае будет достаточно написать только одну перегрузку для условной функции, которая передаст объект дальше:

```cpp
template <typename T>
void f(T x) {
	std::move(x);
}
```

В такой реализации только пользователь определяет, что должно копироваться, а что нет.
Если вызывался от lvalue, то будет копирование, если от rvalue, то будет перемещение.

В векторе же 2 перегрузки, т.к. объекты в нем могут не иметь семантики перемещения.
