Компиляция - перевод из кода понятного человека на машинный язык (бинарники).
## Почти все программы состоят из нескольких файлов

Правильно разделять файлы по логике использования так как:
-  Упрощение повторного использования
-  Упрощение редактирования
-  Упрощение поиска
 Разделяют 3 вида файлов: .cpp - исходный файл, h/.hpp
 -  .cpp - исходный код.
 -  .hpp - хедер содержащий template-ы
 -  .h - обычный хедер с функциями

### Хедеры
Заголовочные файлы, которые содержат в себе только объявление функций, глобальных констант, определение классов и т.д.

```cpp
// math.h

#pranma once

int kSomeConst = 100;
int kAnotherConst = 200;

int Sum(int a, int b);

int Div(int a, int b);

```

### Файл исходников
В файле исходников содержится определение функций для данного заголовка

```cpp
// math.cpp

int Sum(int a, int b) {
	return a + b;
}

int Div(int a, int b) {
	return a - b;
}
```

## Компиляция и препроцессинг

Этапы трансляции:
-  Препроцессиг
-  Компиляция
-  Асемблирование
-  Линковка

Единица трансляции - каждый исходный файл на этапе препроцессинга.

Также каждый компилятор с помощью аргументов может запускать каждый этап трансляции отдельно.
### Препроцессинг
Отдельный язык встроенный в C++, который лексически анализирует код.
Все команды препроцессора называются директивами. А директивы это все команды начинающиеся с #

```cpp
#include <cstdio>
```

В данном случае препроцессор подставит на место cstdio все объявленное внутри библиотеки в том числе и код определений.

После себя препроцессор оставляет все еще .cpp файл
##### Команды препроцессора
Кроме уже названного # include
-  # if
-  # pragma
#### Макросы
Кастомные инструкции препроцессора.

Препроцессор позволяет объявить через директивы некоторые функции. В таком случае объявленный код будет подставлен в место вызова.

```cpp
#include <iostream>

#define MAX(x, y) (x > y ? x : y)

int main() {
	std::cout << MAX(10, 20) << '\n';
	return 0;
}
```

#### Стражи
В том случае, если объявление нескольких функций пересекается в нескольких хедерах. Важно чтобы препроцессор не определил 2 раза одну и ту же функцию иначе будет ошибка компиляции, т.к. определение может быть только одно.

Два случая
```h
#ifndef HEADER_MATH_H
#define HEADER_MATH_H

#endif
```

```h
#pragma once
```

Оба случая предотвращают повторное определение из заголовков.
### Компиляция
Компилируется каждый файл исходника и на выходе получаем транслированный асембленый код.

### Асемблирование
Полученный асемблерный код после компиляции транслируется в машинный код и на выходе получается object файл (файл с машинными инструкциями непосредственно).
Но это еще не готовый файл для запуска. Дело в том, что когда происходит include, то подгружаются только объявленные функции, определения лежат в другом object файле, а на месте вызова неопределенных функций ставятся заглушки.

Каждый полученный object файл называется единицей трансляции.
### Линковка
Заключительная стадия трансляции. 
Задача линковщика заключается в том, что бы найти определения функций по названию в object файлах и подставить в места вызова.
В линковщик заранее записаны адреса, где искать определения всех вызываемых функций.
На выходе из всех единиц трансляции получается полноценный исполняемый файл.

Линковщик очень полезен тем, что он собирает определения из уже скомпилированных библиотек и заново каждый раз их компилировать не нужно.

## Ошибки компиляции
### Ошибка компиляции
Происходит из-за ошибки синтаксиса :)
Не объявлена переменная там или присвоил инту строку, ты ж питонист.

### Ошибки линковки
Если не найдены объявления или определения чего-либо.
Например если компилировать только main.cpp, а файл исходников игнорировать.

## Как ловить UB?
У каждого компилятора есть свой санитайзер, который навешивает на код дополнительные проверки на обращение к памяти.
Очень полезная команда, но позволяет отловить UB только в рантайме.
Пример для g++:

```bash
g++ -fsanitize=address main.cpp
```

Это самый простой санитайзер. Позволяет отловить где процесс обращается не к своей памяти (выведет строку где конкретно). 

Этот вариант санитайзера позволит отловить утечку памяти:

```bash
g++ -fsanitize=leak main.cpp
```

А этот покажет где произошло переполнение типа:

```bash
g++ -fsanitize=undefined main.cpp
```
