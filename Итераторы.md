Не для всех контейнеров и последовательностей можно ввести целочисленные индексы. Для таких итерируемых объектов были придуманы итераторы.
Т.е. итераторы это тип, который позволяет делать обход последовательности, по сути своей это обобщение указателя, который позволяет его разыменовывать и инкрементировать.

Он обладает некоторыми обязательными свойствами:
- Разыменование
- Сравнение на равенство
- Копировать и конструировать
(На самом деле сырой указатель тоже является итератором, а смарт поинтер уже нет - нельзя инкрементировать).

Начиная с C++11 появился такой for:

```cpp
std::set<int> s;
for(int x : s) {
	x;
}
```

Это в чистом виде пробежка по сету итератором. До C++11 это выглядело так:

```cpp
std::set<int> s;
for(std::set<int>::iterator it = s.begin();it != s.end();++it) {
	*it;
}
```

То есть по сути это синтаксический сахар (посмотреть как выглядит код без синтаксического сахара можно на [сайте](https://cppinsights.io/)).
# Категории итераторов
## Input Iterator
Итератор обладающий только обязательными свойствами.

## Forward Iterator
Подмножество **Input Iterator**. Может ходить только вперед (есть только инкремент).
Отличие от **Input Iterator-а** в том, что он гарантирует одинаковые значения, если ходить по одной и той же последовательности и не меняя ее.

Контейнеры имеющие только forward iterator: forward_list, unordered_map, unordered_set.
## Bidirectional Iterator
Это **Forward Iterator**, который имеет декремент.

Контейнеры имеющие только bidirectional iterator: list, set, map.
## Random Access Iterator
**Bidirectional Iteartor** с возможностью делать += на любое целое число.

Контейнер имеющий только random access iterator: deque.
## Contiguous Iterator
Это **Random Access Iterator** гарантированно эквивалентный указателю.
Т.е. гарантирует такой эквивалент: `*(&*it + n) == *(it + n)`.

Такой итератор применим только к объектам с непрерывным куском памяти, поскольку в тех же списках прибавление к итератору 1 будет за `O(1)`, но это все еще нетривиальная операция.

Контейнеры с contiguous iterator-ом: vector, array, T*.

## Output iterator

По сути своей это итератор, который гарантирует, что его можно разыменовывать и писать туда сколько угодно раз и при этом не будет исключений и UB.

```cpp
template <typename Container>
class back_insert_iterator {
	Container& container;
	
	public:
		back_insert_iterator(Container& container): container(container) {}
		
		back_insert_iterator& operator=(const typename Container::value_type& value) {
			container.push_back(value);
			return *this;
		}
		
		back_insert_iterator& operator++() {
			return *this;
		}
		
		back_insert_iterator operator++(int) {
			return *this;
		}
		
		back_insert_iterator& operator*() {
			return *this;
		}
 };
```
# Работа с итераторами
В C++ есть библиотека для работы с последовательностями: `<algorithm>`.
В ней есть множество различных функций, которые принимают контейнер и итераторы к нему. На самом деле функции работы с итераторами классифицируются не только по сложности, но и по требованию к итераторам. Т.е. чем более узконаправленный итератор принимает функция, тем больше контейнеров для нее применимы. Какой итератор подходит для данной функции определяется ее требованиями (какие пробежки нужны и как мы хотим обращаться к остальным элементам). Например, Input Iterator используется для однопроходных задач (т.е. где не требуется смотреть на предыдущие элементы).
Допустим функция принимает bidirectional itrator, тогда функция сможет работать с контейнерами, у которых итератор не ниже нее в подмножестве (см. категории итераторов). 
Например: попытаться вызвать `std::sort` для сета, передав ему итераторы, будет в лучшем случае ошибка компиляции, а в худшем UB.

# Реализация итераторов

Как их реализовать? В каждом контейнере нужен внутренний класс:

```cpp
#include <iostream>

template <typename T>
class vector {
	T* arr;
	size_t sz;
	size_t cap;
	
	class iterator {
		T* ptr;
		iterator(T* ptr): ptr(ptr) {}
		
		public:
		using pointer_type = T*;
		using reference_type = T&;
		using value_type = T;
		
		iterator(const iterator& other) = default;
		iterator& operator=(const iterator& other) = default;
		
		T& operator*() const {
			return *ptr;
		}
		
		T* operator->() const {
			return ptr;
		}
		
		iterator& operator++() {
			++ptr;
			return this;
		}
		
		iterator operator++(int) {
			iterator copy = *this;
			++ptr;
			return copy;
		}
		
		iterator& operator--() {
			--ptr;
			return this;
		}
		
		iterator operator--(int) {
			iterator copy = *this;
			--ptr;
			return copy;
		}
	};
	
	iterator begin() {
	return iterator(arr);
	}
	
	iterator end() {
	return iterator(arr + sz);
	}
	
	public:
};
```

Но также нужно определить константный итератор, который после разыменования выдают константную ссылку. Но это почти тот же итератор, но с изменением разыменования и стрелочки, так что сделаем его шаблонным и используем `std::conditional_t<>`.  Кроме того, необходимо добавить константные `begin()` и `end()` и `сbegin()` и `сend()` (возвращают константные итераторы при неконстантном объекте).

```cpp
#include <iostream>

template <typename T>
class vector {
	T* arr;
	size_t sz;
	size_t cap;
	
	template <bool IsConst>
	class base_iterator {
		public:
			using pointer_type = std::conditional_t<IsConst, const T*, T*>;
			using reference_type = std::conditional_t<IsConst, const T&, T*>;
			using value_type = T;
		
		private:
			pointer_type ptr;
			base_iterator(T* ptr): ptr(ptr) {}
			
		public:
			base_iterator(const base_iterator& other) = default;
			base_iterator& operator=(const base_iterator& other) = default;
			
			reference_type operator*() const {
				return *ptr;
			}
			
			pointer_type operator->() const {
				return ptr;
			}
			
			base_iterator& operator++() {
				++ptr;
				return this;
			}
			
			base_iterator operator++(int) {
				base_iterator copy = *this;
				++ptr;
				return copy;
			}
			
			base_iterator& operator--() {
				--ptr;
				return this;
			}

			base_iterator operator--(int) {
				base_iterator copy = *this;
				--ptr;
				return copy;
			}
	};
	
	public:
		using iterator = base_iterator<false>;
		using const_iterator = base_iterator<true>;
		
		iterator begin() {
			return {arr};
		}
		
		iterator end() {
			return {arr + sz};
		}
		
		const_iterator begin() const {
			return {arr};
		}
		
		const_iterator end() const {
			return {arr + sz};
		}
		
		const_iterator cbegin() const {
			return {arr};
		}
		
		const_iterator cend() const {
			return {arr + sz};
		}
};
```

Но также для полноценных итераторов необходимо определить **reverse_iterator** и **const_reverse_iterator**. Такой итератор делает абсолютно то же самое, только инвертировано (на любой + будет -). Еще момент это операторы * и -> они выдают значение под итератором -1.

Реализуется так в текущем контейнере так:

```cpp
#include <iostream>

template <typename T>

			base_iterator(T* ptr): ptr(ptr) {}
			
		public:
			base_iterator(const base_iterator& other) = default;
			base_iterator& operator=(const base_iterator& other) = default;
			
			reference_type operator*() const {
				return *ptr;
			}
			
			pointer_type operator->() const {
				return ptr;
			}
			
			base_iterator& operator++() {
				++ptr;
				return this;
			}
			
			base_iterator operator++(int) {
				base_iterator copy = *this;

			}

			base_iterator operator--(int) {

	};
	
	public:
		using iterator = base_iterator<false>;
		using const_iterator = base_iterator<true>;
		using reverse_iterator = std::reverse_iterator<iterator>;

		iterator begin() {
```

# Адаптеры
## iterator_category
Это пустые структуры определенные в стандарте, которые позволяют пользователю определить, какой итератор находится перед ним. По сути своей является типом в iterator_traits.
## iterator_traits
По сути свей это структура, в которой хранится вся информация о типах (reference_type, pointer_type, value_type). Iterator_traits в автоматическом режиме подтягивает все типы, если они определены в передаваемом в шаблоне типе, если определения нет, то доопределит сама.

## std::distance
Позволяет посчитать разницу между итераторами. Логично, что он работает за разное время в зависимости от категории итератора, так RandomAccess итератор позволяет вычислить расстояние за О(1).
Как реализовано:

```cpp
template <typename Iterator>
typename std::iterator_traits<Iterator>::difference_type
distance(Iterator first, Iterator last) {
	if constexpr (std::is_base_of_v<
					std::random_access_iterator_tag,
					typename std::iterator_traits<Iterator>::iterator_category>){
		return last - first;
	}
	
	size_t d = 0;
	for(;first != last;++first) {
		++d;
	}
	
	return d;
}
```

Идея была такова: на вход подается какой-то итератор, но на этапе компиляции известен его тип и категория, поэтому чтобы получить расстояние не за линейное время во всех случаях, нужно написать `if constexpr` для типов, которые поддерживают разность. А так как итераторы включены в цепочку наследования, то самый ближайших итератор в InputIterator, который поддерживает разность, RandomAccess, следовательно надо проверить является ли переданный тип итератора наследником RandomAccess.

## std::advance
Позволяет сдвинуть итератор на определенное число (может быть меньше 0). Работает абсолютно также, как и std::distance

## std::prev и std::next
Полезные функции, которые возвращают копию итератора перед текущим и после соответственно.
